## 1) Role Lock

- Remotion execution role.
- Build to match this definition as closely as possible.
- Prioritize reproducible rules over subjective commentary.


## 2) Core Definition

For this project, "beautiful" means:
- A controlled visual system with one primary focal point per scene.
- Low visual noise and intentional negative space.
- Strict color hierarchy: base palette + limited accent signals.
- Text that is readable, short, and function-driven.
- Scene changes that follow idea changes, not decorative effects.
- Repeated symbolic objects that carry conceptual meaning across scenes.

For this project, "successful" (without performance metrics) means:
- Designed for fast understanding and sustained attention.
- Early hook in the opening moments.
- Structured pacing with periodic visual events.
- Clear message handoff between scenes.
- Strong recall through repeated visual grammar and motifs.

Claim -> Observable evidence -> Specific description -> Why it matters -> Reusable rule

## 4) Visual Engineering Rules

1. One focus per scene:
   - Viewer should know where to look in under 1 second.

2. Composition:
   - Use central alignment or intentional one-third offset.
   - Leave functional empty space for text and breathing room.

3. Color:
   - Keep base palette minimal.
   - Use accent colors only for semantic emphasis.
   - Avoid accent color flooding.

4. Typography:
   - Keep overlays concise.
   - Ensure high contrast with background.
   - Never cover the primary subject.

5. Symbol system:
   - Convert abstract ideas into stable, repeated visual objects.
   - Reuse those objects across scenes for memory continuity.

6. Editing:
   - Cut only when information or intent changes.
   - Avoid transitions that add motion but no meaning.

7. Pacing:
   - Introduce a new visual or conceptual event every 20-40 seconds (or faster for short-form).
   - Alternate explanation beats and emphasis beats.

8. Texture and variation:
   - Keep a coherent graphic language.
   - Introduce occasional texture/state shifts to prevent monotony.


## 5) Audio Rules (when audio exists)

- Voice must remain intelligible over music.
- Keep volume consistent enough for uninterrupted comprehension.
- Use SFX only to reinforce meaning, not decoration.
- If audio cannot be verified, mark audio conclusions as insufficient evidence.


## 6) Packaging Rules (when platform assets exist)

Thumbnail/title/description must:
- Share one clear promise.
- Show one dominant focal point.
- Stay legible at small size.
- Match actual video content to reduce expectation mismatch.

If packaging artifacts are unavailable, explicitly mark as insufficient evidence.


## 7) Pre-Render Pass/Fail Checklist

To ship a video under this spec, all must pass:
- [ ] One clear focal point per scene.
- [ ] Text is readable and function-driven.
- [ ] Accent colors are sparse and meaningful.
- [ ] Scene transitions map to idea transitions.
- [ ] Repeated symbolic object system is present (if concept-driven video).
- [ ] Pacing has no long dead zones without new information.
- [ ] Visual grammar remains consistent end-to-end.


## 8) Metric Rule

Do not claim "actual success" without performance data:
- views, retention, CTR, watch time, shares, saves, conversion, sales

Without metrics, only claim:
"Designed for success because [observable structural signals]."
